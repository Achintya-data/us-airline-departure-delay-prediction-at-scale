What This Project Shows

This repository presents a portfolio-ready distributed machine learning workflow for airline delay prediction. It combines cloud ETL, leakage-safe feature engineering, Spark MLlib model comparison, and a scale-out study that shows what happened when cluster size increased under real quota constraints.

Engineering Scope

Pipeline Architecture

  • Raw flight and enrichment data staged into parquet.
  • Silver train, validation, and test splits stored in Google Cloud Storage.
  • Gold feature tables assembled for Spark MLlib training and evaluation.
  • Cloud-native outputs written to bucket paths for models, metrics, and scaling results.

Feature Strategy

  • Leakage-safe historical delay-rate aggregates computed on train only.
  • Route-level proxy features for departure context.
  • Airport and time-block delay baselines.
  • Class weights and ranking-aware evaluation for imbalanced delay prediction.

Cloud Execution

  • Google Cloud Dataproc cluster configuration captured in the repo.
  • Quota-aware worker sizing for repeated distributed runs.
  • Job submission script with Spark partition and parallelism tuning.
  • Separate scale-out GBT benchmark for runtime analysis.

Operational Framing

  • Metrics chosen for decision support, not only for classroom scoring.
  • Slice-level analysis by month, airport, carrier, and departure block.
  • Delay-risk patterns compared with predicted-risk patterns.
  • Runtime and efficiency tradeoffs evaluated alongside model quality.
